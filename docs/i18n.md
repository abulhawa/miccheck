# i18n scaffolding

## Adding a locale
1. Add a new JSON file in `apps/web/lib/i18n/locales/` (for example, `fr.json`).
2. Add the locale to the `Locale` union and `localeMaps` in `apps/web/lib/i18n/index.ts`.
3. Keep keys flat and aligned with the naming conventions below.

## Adding keys
1. Add the key/value pair in `apps/web/lib/i18n/locales/en.json`.
2. Optionally add the translated string to the other locale files.
3. Use `t("section.subsection.label")` in the UI (and `resolveCopy` where the rest of the copy system is used).

## Key naming conventions
- Use `section.subsection.label` format.
- Examples:
  - `level.acceptable_noise_first`
  - `advice.noise_fix_before_gain`
  - `affiliate.disclosure_short`

## Missing keys in development
In development mode, missing translation keys log a console warning so you can fill them in before shipping.
